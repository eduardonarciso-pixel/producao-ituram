Documentação técnica — Painel Produção Ituram

Visível apenas para o perfil Administrador. Descreve o que foi construído, as fórmulas de cálculo e como manter o painel.

1. O que é

Painel web para lançamento e acompanhamento diário da produção de motos das equipes A e B. Um único arquivo (index.html) publicado no GitHub Pages, sem servidor próprio: toda a lógica roda no navegador e os dados ficam num banco Supabase (Postgres) compartilhado por todos que acessam o link.

2. Perfis e permissões

PerfilConsultarLançarEditar dia existenteExcluir
AdministradorSimSimSimSim
ArmazémSimSim (dia novo)NãoNão
VisitanteSimNãoNãoNão

A permissão é aplicada em dois níveis: na tela (esconde formulário e botões para quem não pode usá-los) e, o que realmente importa para a segurança, no banco de dados via Row Level Security do Supabase — ou seja, mesmo alguém lendo o código-fonte da página não consegue gravar nada fora do que o perfil permite. As senhas ficam no Supabase (Authentication → Users), nunca neste arquivo.

3. Memória de cálculo dos indicadores

Motos produzidas no períodoSoma de prod de todos os lançamentos dentro do período filtrado.
Média por dia trabalhadoSoma total ÷ quantidade de dias com prod > 0 (dias com feriado/parada em 0 não entram na média).
Recorde diárioLançamento de maior prod no período filtrado.
Dias na metaQuantidade de dias com prod ≥ META (META = 50, configurável no início do arquivo) sobre o total de dias lançados.
Comparativo por equipeMesmas três fórmulas acima (total, dias, média), aplicadas separadamente aos lançamentos de cada equipe.
Gráfico diárioUma barra por lançamento; altura proporcional a prod. Escala vertical = maior valor entre 100 e o próximo múltiplo de 10 acima do maior lançamento do período. Linha tracejada vermelha na altura da meta. ◆ marca dias com observação preenchida.
Gráfico mensalLançamentos agrupados por competência (ano-mês); barra empilhada com o total da Equipe A embaixo e Equipe B em cima. O número acima da barra é a soma do mês.
Filtro de períodoRecalcula tudo acima (KPIs, gráfico diário, comparativo) considerando apenas data ≥ início e data ≤ fim quando preenchidos. O gráfico mensal e a tabela de lançamentos não são afetados pelo filtro — mostram sempre o histórico completo.

4. Regra de negócio: um lançamento por dia

A data é chave única na tabela producao. Armazém só consegue inserir data nova; se a data já existir, o banco recusa (o painel avisa "apenas o administrador pode alterá-lo"). O Administrador tem permissão de substituir (upsert) um dia já lançado — é o que acontece ao clicar em "Editar" na tabela: o formulário é preenchido com os dados do dia e, ao salvar, o registro é atualizado.

5. Notificação por e-mail

Um gatilho no banco de dados (função Postgres + extensão pg_net) dispara sempre que o perfil Armazém insere um novo lançamento, enviando um e-mail (via API do Resend) para eduardo.narciso@supplog.com com data, equipe, quantidade, se bateu a meta e a observação. Por decisão do administrador, edições e exclusões feitas pelo perfil Administrador não geram e-mail.

6. Arquitetura e arquivos do projeto

index.htmlO painel inteiro (HTML + CSS + JavaScript puro, sem frameworks). É o único arquivo publicado no site.
setup-supabase.sqlCria a tabela producao e importa o histórico original da planilha.
setup-acessos.sqlDefine as regras de permissão (Row Level Security) por perfil.
setup-email.sqlCria o gatilho de notificação por e-mail. Contém uma chave de API — nunca deve ser enviado ao repositório público do GitHub.
LEIA-ME.mdGuia de configuração inicial (Supabase, publicação, perguntas frequentes).

Gráficos (diário e mensal) são desenhados em SVG puro, sem biblioteca externa. Dados trafegam diretamente entre o navegador e a API REST do Supabase (fetch), autenticados com o token de sessão do login. A atualização é automática a cada 60 segundos e sempre que a aba volta ao foco.

7. Publicação e manutenção

Site: github.io/producao-ituram (repositório público eduardonarciso-pixel/producao-ituram, GitHub Pages). Para publicar uma alteração: Add file → Upload files no repositório, subir o index.html atualizado por cima do existente e confirmar o commit — o site atualiza em cerca de 1 minuto. Backup/exportação dos dados: Supabase → Table Editor → tabela producao → Export → CSV.
